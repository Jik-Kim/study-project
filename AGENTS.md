# AGENTS.md

이 파일은 저장소 전체에 적용되는 공통 작업 규칙이다. 하위 디렉터리의
`AGENTS.md`에는 해당 영역에만 필요한 규칙을 둔다.

## 작업 시작

- 작업 전에 `docs/SOT.md`, 현재 `git status`를 확인한다.
- `docs/feedback.local.md`가 있으면 개인 작업 피드백으로 함께 확인한다.
- 아키텍처는 `docs/architecture.md`, 통신 계약은 `docs/interfaces.md`를 참고한다.
- 빌드와 실행 방법은 `docs/setup.md`를 기준으로 한다.
- 문서와 코드가 충돌하면 승인된 최신 SOT를 우선하고 충돌 사실을 알린다.
- 기존 사용자 변경 사항과 관련 없는 파일을 수정하거나 되돌리지 않는다.

## 작업 범위

- 요청받은 범위에서 가장 작은 변경으로 목적을 달성한다.
- 패키지 경계나 기술 체계를 크게 바꿀 때는 변경 구조를 먼저 제안하고 검토받는다.
- 확정되지 않은 설계, 파라미터, 메시지 필드를 임의로 결정하지 않는다.
- 구현을 요청받지 않은 영역은 한글 TODO로 남긴다.
- 새로운 외부 의존성, 패키지 경계, 공통 인터페이스 변경은 먼저 사용자에게 알린다.
- 요청하지 않은 git push, PR 생성은 수행하지 않는다.

## 프로젝트 구조

- `gesture_robot_interfaces`는 rosidl 인터페이스 전용 `ament_cmake` 패키지다.
- `gesture_robot`은 rclpy 애플리케이션 `ament_python` 패키지다.
- 노드 간 데이터는 직접 호출이 아니라 합의된 ROS2 인터페이스로 전달한다.
- 대용량 이미지와 영상은 Git에 커밋하지 않는다.

## 코드 규칙

- 함수 하나는 하나의 책임을 갖도록 작성한다.
- 변경 가능한 실행 값은 코드에 하드코딩하지 않고 ROS2 파라미터로 관리한다.
- 설명이 필요한 주석과 TODO는 한국어로 작성한다.

## 문서 동기화

- MVP, 패키지 구조, 기술 방향이 바뀌면 `docs/SOT.md`를 갱신한다.
- 코드와 문서가 불일치하면, 코드를 최신 정보로 간주한다.
- 메시지나 토픽 계약이 바뀌면 `docs/interfaces.md`와 SOT를 함께 갱신한다.
- 빌드 또는 실행 절차가 바뀌면 `docs/setup.md`를 갱신한다.
- 회의록 원문은 Notion에서 관리하고 확정된 결정만 SOT에 반영한다.

## 검증과 완료 보고

- 변경한 범위에 맞는 가장 작은 검증부터 실행한다.
- ROS2 패키지 구조 변경 후 `colcon build --symlink-install`을 확인한다.
- 검증하지 못한 항목은 성공한 것처럼 표현하지 않고 이유를 명시한다.
- 완료 시 변경 내용, 검증 결과, 남은 TODO를 간결하게 보고한다.
